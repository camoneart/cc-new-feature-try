# cc-new-feature-try

Claude Code の新機能を試すためのサンドボックスプロジェクト。

## purpose

- Claude Code のアップデートで追加された新機能の動作確認

## 参照

- [Claude Code Changelog](https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md)

## issue化の判断基準

changelog の項目を GitHub issue にするかどうかは、以下の基準で判断する。

### issue化する（= 手を動かして試せて、学びがあるもの）

- ユーザーが直接操作・設定できる機能（コマンド、フラグ、設定項目）
- 新しいフックや環境変数が追加された場合
- UIの変化が目に見える変更（スピナー、ステータスラインなど）
- ワークフローが変わる大きな機能（Agent Teams、Worktree統合など）
- 以前自分が困っていたバグの修正確認

### issue化しない（= 試せない、または学びが薄いもの）

- 内部のメモリリーク修正やパフォーマンス最適化
- 内部リファクタリングやコード整理
- 依存関係のバージョンアップ
- ユーザーに見えない内部キャッシュ改善

### 優先度

候補が多い場合は以下で優先度を付ける

- **High**: 日常のワークフローに直接影響する機能
- **Medium**: 知っておくと便利な新コマンド・新設定
- **Low**: ニッチだが面白い特殊なフック・環境変数
