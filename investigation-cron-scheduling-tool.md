# 調査結果: cron スケジューリングツール (v2.1.71)

## 概要

v2.1.71 で追加された cron スケジューリングツール（CronCreate, CronList, CronDelete）について調査した。

## 追加されたツール

| ツール | 役割 |
|--------|------|
| CronCreate | 新しいスケジュールを作成 |
| CronList | アクティブなスケジュールを一覧表示 |
| CronDelete | スケジュールを削除 |

## `/loop` コマンドとの関係

- `/loop` はユーザーが直接使うスラッシュコマンド（ユーザー向けインターフェース）
- CronCreate/CronList/CronDelete は Claude が内部的に使用するツール（ツールAPI）
- `/loop` の内部実装として cron ツールが使われている

### 使い分け

| 観点 | `/loop` | cron ツール |
|------|---------|------------|
| 使用者 | ユーザー | Claude 自身 |
| 呼び方 | `/loop 5m "git status確認して"` | Claude が判断して自動的に使う |
| 抽象度 | 高い（簡単に使える） | 低い（細かい制御が可能） |
| 起動の主体 | 常にユーザー | Claude が自律的に判断可能 |

### 実用上の違い

- `/loop`: 明示的にコマンドで「これを繰り返せ」と指示する命令的なアプローチ
- cron ツール: 「5分ごとにgit status確認して」のような自然言語の依頼から Claude が自分で CronCreate を使ってスケジュールを組める
- cron ツールの追加により、ユーザーは `/loop` コマンドを知らなくても自然言語で定期実行を依頼できるようになった

## スコープ

- OS レベルの crontab ではなく、Claude Code セッション内に限定されたスケジューリング
- セッションを閉じるとスケジュールも消える

## 所感

- `/loop` と cron ツールは機能的にはほぼ同等だが、誰が起動するか（ユーザー vs Claude）という点で異なる
- cron ツールの追加により、Claude の自律性が向上した（自分の判断で定期実行を設定できる）
- 自然言語で「定期的にやっておいて」と依頼するだけで済むため、ユーザー体験が向上している
