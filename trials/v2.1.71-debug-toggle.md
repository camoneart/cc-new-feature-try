# /debug トグル更新 - 調査結果

## 対象バージョン

- Claude Code v2.1.71（ビルド: 2.1.71.752）

## 変更概要

v2.1.71 から、デバッグログがデフォルトで書き込まれなくなった。
`/debug` コマンドがセッション中にデバッグログのオン/オフを切り替える動作に更新された。

## 試行結果

### 1. デバッグログの有効化

- `/debug` を実行すると、デバッグログが有効化された
- ログファイルは `~/.claude/debug/<session-uuid>.txt` に作成される
- `/debug` 実行前のログは記録されない（デフォルトオフを確認）

### 2. ログの記録確認

- `/debug` 有効化後、ツール実行（Read等）のログが正常に記録された
- タイムスタンプはミリ秒単位（ISO 8601形式）
- ログレベルは `[DEBUG]` のみ確認（`[ERROR]`/`[WARN]` はなし）

### 3. 2回目の `/debug` 実行

- 2回目の `/debug` 実行後もログは継続記録された
- ログサイズは 46.2KB に成長
- セッション内で一度有効化すると、セッション終了まで有効のまま

### 4. 起動時からのログ取得

- セッション途中からの有効化: `/debug` コマンド
- 起動時からの有効化: `claude --debug` フラグ

## デバッグログの内容

ログから以下の情報が確認できる：

| カテゴリ | 内容 |
|---|---|
| API通信 | リクエスト作成、OAuth認証、ストリーム開始 |
| Hooks | Stop/PreToolUse フック実行、approve/deny 判定 |
| ツール実行 | PreToolUse/PostToolUse のフック呼び出し |
| コンテキスト管理 | トークン使用量、autocompact 閾値 |
| 自動更新 | バージョンチェック、インストール状態 |
| セッション管理 | FileHistory スナップショット、caffeinate |
| ファイルI/O | アトミック書き込み（tmp + rename） |

## 所感

- デフォルトオフへの変更は、通常使用時のディスクI/Oオーバーヘッド削減に有効
- トラブルシューティング時に `/debug` で即座に有効化できるのは便利
- ログの粒度が細かく、API遅延やHookの動作確認に活用できる
- `managed-settings.json` の ENOENT は個人利用では正常（エンタープライズ管理用ファイル）
